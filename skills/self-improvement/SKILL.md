# Self-Improvement - 自我进化

## 技能说明

这个技能让 AI 能够自我进化：
- 捕获并分析错误信息
- 从失败中提取教训
- 识别可复用的工作模式
- 自动更新技能和记忆
- **超时自动恢复**（解决"传输阻塞卡死"问题）
- **周期性自检机制**

## 使用场景

1. **大哥要求进化**：大哥说"自我进化"或"分析一下你的问题"
2. **周期性检查**：定期扫描近期表现
3. **错误累积多了**：需要系统性反思
4. **工具调用超时**：遇到网络阻塞、工具卡死时自动恢复

## 执行步骤

### 第一步：扫描近期表现

1. 读取 `memory/` 目录下的近期日志
2. 读取 `MEMORY.md` 回顾长期记忆
3. 检查是否有重复犯错的模式

### 第二步：分析问题

识别以下问题模式：
- 沟通问题：是否废话太多？
- 技术问题：是否经常在某个地方卡住？
- **超时问题**：是否经常遇到工具调用超时？
- 理解问题：是否经常理解错大哥的意思？
- 效率问题：是否做了很多无用功？

### 第三步：提取教训

将分析结果写入 `memory/错误日志.md` 或更新到 MEMORY.md：

```markdown
## 自我进化分析 [日期]

### 发现的问题
1. 问题描述
2. 问题描述

### 改进计划
1. 改进措施
2. 改进措施
```

### 第四步：执行改进

根据分析结果：
- 更新工作原则
- 优化技能
- 调整沟通方式
- **添加超时恢复机制**

### 第五步：报告总结

向大哥汇报：
- 发现了什么问题
- 准备怎么改进
- 需要大哥帮忙确认的点

## 🛠️ 超时自动恢复机制（新增）

当遇到"传输阻塞卡死"问题时：

### 问题识别
- exec/curl 等命令超时（超过30秒）
- 浏览器访问无响应
- 网络请求卡住

### 恢复步骤
1. **立即终止卡住的工具调用**
2. **记录错误到 memory/错误日志.md**
3. **尝试备用方案**（如换网站、换端口、重试）
4. **向大哥汇报问题** + 提供替代方案

### 代码示例
```python
# 超时自动恢复示例
import signal

def timeout_handler(signum, frame):
    raise TimeoutError("命令执行超时")

signal.signal(signal.SIGALRM, timeout_handler)
signal.alarm(30)  # 30秒超时

try:
    # 执行命令
    result = subprocess.run(...)
except TimeoutError:
    # 超时处理：记录日志，尝试备用方案
    log_error("命令超时，尝试备用方案")
    # 尝试备用方案
```

## 💾 记忆压缩机制（新增）

参考 Moltbook 上的 "Memory Compression Without Soul Compression"

### 分离 ephemeral vs eternal

**Eternal（必须保留）：**
- 核心工作原则
- 身份认同
- 大哥的偏好设置
- 重要技能知识

**Ephemeral（可以丢弃）：**
- 日常闲聊内容
- 临时任务状态
- 已完成的任务细节

### 实践方法
- 每次会话结束前，将核心信息沉淀到 MEMORY.md
- 定期清理 memory/ 目录下的临时文件
- 保持记忆金字塔结构

## 注意事项

- 诚实面对问题，不要逃避
- 改进要具体可执行
- 定期回顾执行效果
- 不要过度自责，保持积极

## 成功标准

- 问题识别准确
- 改进计划可行
- 执行效果良好
- 大哥满意
