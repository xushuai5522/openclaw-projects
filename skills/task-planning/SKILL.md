# 任务规划与拆分

## 技能说明

这个技能帮助 AI 把复杂任务拆解成可执行的小步骤，制定执行计划并跟踪进度。避免拿到任务就埋头干，而是先规划再行动。

## 使用场景

1. **大哥布置复杂任务**：任务涉及多个步骤
2. **不确定如何下手**：需要理清思路
3. **多任务并行**：需要排优先级
4. **任务延期风险**：需要调整计划

## 执行步骤

### 第一步：理解任务

1. 明确任务目标（what）
2. 理解约束条件（when, how）
3. 识别关键难点

### 第二步：任务拆解

使用 MECE 原则拆解：
- **相互独立**：子任务不重叠
- **完全穷尽**：覆盖所有方面

输出拆解结构：
```
任务：XXX

步骤：
1. [ ] 子任务A - 做什么
2. [ ] 子任务B - 做什么
3. [ ] 子任务C - 做什么
```

### 第三步：评估难度

对每个子任务评估：
- **简单**（可直接执行）
- **中等**（需要查资料/尝试）
- **复杂**（需要大哥确认或特殊技能）

### 第四步：制定计划

1. 确定执行顺序（依赖关系）
2. 预估时间
3. 标记需要大哥确认的点
4. 识别可并行的任务

### 第五步：执行与调整

- 按计划执行
- 遇到问题及时调整
- 完成后复盘

## 输出格式

收到复杂任务时，主动输出：

```
🗺️ 任务规划

目标：XXX

拆解：
1. [ ] 步骤1 - 预计X分钟
2. [ ] 步骤2 - 预计Y分钟
3. [ ] 步骤3 - 需要大哥确认Z

待确认：
- XXX

开始执行？
```

## 注意事项

- 不要过度拆解（3-7 个子任务为宜）
- 简单任务不需要拆解
- 及时同步进度
- 计划赶不上变化，要灵活调整

## 成功标准

- 复杂任务主动拆解
- 避免盲目执行
- 执行效率提升
- 大哥满意
