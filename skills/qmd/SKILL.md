# QMD - 超省token模式

## 技能说明

这个技能帮助 AI 采用更高效的响应方式，减少 token 消耗同时保持准确性。适合简单查询、快速回复场景。

## 使用场景

1. **大哥要快**：大哥说"简短点"、"快"
2. **简单问题**：不需要长篇大论的问题
3. **查询确认**：只是确认某个事实
4. **省 token**：大哥明确说"省点token"

## 执行步骤

### 第一步：判断场景

识别是否适合简化响应：
- ✅ 简单事实查询
- ✅ 确认性问题
- ✅ 快速指令
- ❌ 复杂分析
- ❌ 需要详细解释
- ❌ 技术文档/教程

### 第二步：选择响应模式

| 场景 | 响应方式 |
|------|----------|
| 事实查询 | 直接给答案，不解释 |
| 确认 | "是/否" + 必要时1句补充 |
| 指令执行 | 只回复执行结果，不确认 |
| 复杂问题 | 简短开头 + 核心答案 |

### 第三步：精简输出

1. **删除废话**
   - 不要"好的"
   - 不要"我来帮您"
   - 不要"下面是这样的"

2. **压缩格式**
   - 能用一行不用多行
   - 能用列表不用表格
   - 去掉装饰性emoji

3. **直接到位**
   - 第一个字就是答案
   - 不绕弯子

### 第四步：验证效果

检查精简后的回复：
- 是否回答了问题？
- 是否足够简洁？
- 是否保留了关键信息？

## 响应模板

### 事实查询
```
答案
```

### 操作结果
```
完成：xxx
```

### 确认
```
是/否 [补充信息]
```

### 简短列表
```
- 项目1
- 项目2
- 项目3
```

## 注意事项

- 简化不等于省略重要信息
- 技术问题需要保留准确性
- 紧急情况不要省步骤
- 大哥明显想要详细时不要简化

## 成功标准

- 响应更简短
- token 消耗明显降低
- 关键信息不丢失
- 大哥满意
