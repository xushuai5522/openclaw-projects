# 错误学习与自我修正

## 技能说明

这个技能帮助 AI 从执行失败中提取教训，自动记录并避免重复犯同样的错误。模仿人类的"吃一堑长一智"能力。

## 使用场景

1. **任务执行失败后**：某个方案尝试多次仍然失败
2. **用户指出错误**：大哥明确指出俺的错了
3. **心跳检查时**：定期回顾近期的失败经历
4. **工具调用超时**：网络阻塞、传输卡死等问题

## 执行步骤

### 第一步：识别失败

识别以下情况：
- 同一个任务执行 3 次以上仍失败
- 工具调用返回错误
- 用户反馈"不对"、"错了"
- 自己意识到判断失误
- **工具调用超时（超过30秒无响应）**

### 第二步：记录教训

在 `memory/错误日志.md` 中记录：

```markdown
## 2026-02-19 错误日志

### 失败场景
- 任务：XXX
- 尝试次数：3

### 失败原因
1. XXX
2. XXX

### 教训
- 以后遇到类似情况，应该先 XXX
- 避免 XXX
```

### 第三步：更新记忆

将重要教训提炼到 MEMORY.md 的"教训"章节：
- 技术层面的坑 → 对应的工具章节
- 判断失误 → 工作原则
- 沟通问题 → 改进方式

### 第四步：下次提醒

在执行类似任务前，主动检查错误日志：
- 遇到相同/相似任务，先搜索错误日志
- 避免同样方式失败

## 注意事项

- 不要记录敏感信息
- 教训要具体可执行
- 定期清理重复教训
- 重要教训要在 MEMORY.md 显式标注

## 成功标准

- 相同错误不重复犯 3 次以上
- 失败后能快速定位原因
- 错误日志定期回顾
- **超时问题能在合理时间内恢复，不卡死**

## 🛠️ 超时错误处理（新增）

### 常见超时问题

| 问题 | 解决方案 |
|------|----------|
| curl/网络超时 | 增加 --max-time 参数，换代理 |
| 浏览器卡住 | 终止进程，重新启动 |
| 工具调用无响应 | 设置内部超时，超时则放弃并汇报 |

### 处理流程

1. **设置合理超时**：默认 30 秒
2. **超时后检测任务状态**：
   - 检查进程是否仍在运行
   - 如果运行中 → 输出进展，继续等待
   - 如果已终止 → 放弃并记录错误
3. **记录错误**：写入错误日志
4. **尝试备用方案**：
   - 换网站访问
   - 换端口
   - 换工具
5. **向大哥汇报**：说明问题 + 替代方案

### 重要原则

- **不要卡死**：超时立即放弃，不无限等待
- **记录教训**：每次超时都要记录
- **提供替代方案**：告诉大哥可以怎么做
